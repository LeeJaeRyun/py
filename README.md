## HW1.py
1. 첫 번째 for 루프:
- fast 포인터는 k칸 앞으로 이동합니다.
- slow 포인터는 제자리에 머무릅니다.
- 이제 fast와 slow 사이에는 k 노드만큼의 간격이 생깁니다.
- fast가 리스트의 끝에 도달하려면 앞으로 N−k 개의 노드를 더 지나야 합니다.

2. 두 번째 while 루프:
- fast와 slow는 이제 한 칸씩 동시에 움직입니다.
- fast가 None (리스트의 끝)에 도달할 때까지 움직인 횟수는 N−k 입니다.
- slow는 fast와 보조를 맞춰 움직이므로, slow 역시 N−k 칸을 이동합니다.

3. slow의 최종 위치:
- slow는 첫 번째 노드(위치 1)에서 시작했습니다.
- slow는 총 N−k 칸을 이동했습니다.
- 따라서 slow의 최종 위치는 1+(N−k) 번째 노드입니다.
